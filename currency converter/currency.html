    How it works

    We first check your browser's cache (10‑minute TTL). If fresh, we use that rate. Otherwise we try the primary API (exchangerate.host). If it fails, we fall back to frankfurter.app and then open.er-api.com. Exponential backoff (2 retries) is applied. If all APIs fail but a stale cache exists, you'll see the last known rate with a timestamp. No cache? You'll get an offline message with a retry button.
